1、常用命令

  • 1、git init 初始化本地库
  • 2、git status 查看本地库状态【不在暂存区的文件会显示红色；在暂存区的文件显示绿色】
  • 3、git add 文件名 将文件添加到暂存区
  • 4、git rm --cached 文件名 将暂存区的文件删除掉
  • 5、git commit -m "提示信息" 文件名 将暂存区的文件提交到本地库
  • 6、git reflog 查看精简版本信息
  • 7、git log 查看版本详细信息
  • 8、git reset --hard 版本号 版本穿梭

.git/HEAD文件：说明当前是在哪个分支

.git/refs/heads/master文件：记录着当前的版本号

2、分支操作

  • 1、git branch -v 查看分支
  • 2、git branch 分支名 创建分支
  • 3、git checkout 分支名 切换分支
  • 4、git merge 分支名 把指定的分支合并到当前分支上

冲突合并步骤：

  • 1、首先手动打开发生冲突的文件，把所有不想要的部分统统删除
  • 2、将发生冲突的文件修改完成后，添加到暂存区 ：git add 文件名
  • 3、执行提交（注意：此时使用git commit命令时不能带文件名）：git commit -m "注释"
  • 合并冲突解决后，只会修改当前分支，待合并分支的内容仍然是合并之前的

3、协作命令

  • 1、git remote -v 查看当前所有远程地址别名
  • 2、git remote add 别名 远程地址 给远程仓库起别名
  • 3、git push 别名 分支名 将本地仓库的某个分支上的内容推送到远程仓库
  • 4、git pull 远程库地址别名 远程分支名 将远程仓库的某个分支最新内容拉下来后与当前本地分支直接合并
  • 5、git clone 远程地址 将远程仓库的内容克隆到本地
    • clone 会做如下3件操作：
      • 1、拉取代码；
      • 2、初始化本地仓库；
      • 3、创建别名（注意：在clone自动给创建的别名统一叫：origin）
