反射

1、概述

反射：将类的各个组成部分封装成其它对象

Java代码在计算机中经历的三个阶段：

  • source源代码阶段：.java经过javac 编译成.class字节码文件 ，字节码文件包括java文件中的成员变量、构造方法、成员方法

  • Class类对象阶段：通过类加载器【ClassLoader】将字节码文件加载到内存中，变成Class类对象，Class类对象中包含：成员变量Fields[ ]对象，构造方法Contructors[ ]对象，成员方法Methods[ ]对象

  • Runtime运行时阶段：通过Class类对象中的一些行为创建对象 new 对象()

反射的好处：

  • 可以在程序的运行过程中，操作Class类对象中的这些对象

  • 可以接耦，提高程序的可扩展性

2、获取字节码Class类对象的三种方式

2.1、在source阶段：

此时Java代码只有字节码文件，并没有进内存，需要手动将字节码文件 加载进内存，生成Class类对象

多用于配置文件，将类名定义在配置文件中，读取文件，加载类

2.2、在Class类对象阶段：

此时已经将字节码文件加载进内存了，直接获取Class类对象

多用于参数的传递

2.3、在Runtime运行时阶段：

此时已经有类的对象了，可以通过对象的方法来获取Class类对象

多用于对象的获取字节码的方式

结论：

同一个字节码文件（.class）在一次程序运行过程中，只会被加载一次，不论通过哪一种方式获取的Class类对象都是同一个

3、Class类对象功能概述

带Declared的 可以不考虑修饰符

如果想要获取private 修饰的成员，需要通过暴力反射field/constructor/method.setAccessible(true) 来忽略访问权限修饰符的安全检查

  • 获取功能

    • 1、获取成员变量

      • Field[] getFields()

        • 获取所有public修饰的成员变量

      • Field getField(成员变量名)

        • 获取指定名称的public修饰的成员变量

      • Field[] getDeclaredFields()

        • 获取所有的成员变量（不考虑修饰符）

      • Field getDeclaredField(成员变量名)

        • 获取指定名称的成员变量（不考虑修饰符）

    • 2、获取构造方法

      • Constructor<?>[] getConstructors()

        • 获取所有public修饰的构造方法

      • Constructor<?> getConstructor(参数列表)

        • 获取public修饰的指定的构造方法

      • Constructor<?>[] getDeclaredConstructors()

        • 获取所有的构造方法

      • Constructor<?> getDeclaredConstructor(参数列表)

        • 获取指定的构造方法

    • 3、获取成员方法

      • Method[] getMethods()

      • Method getMethod(方法名,参数列表)

      • Method[] getDeclaredMethods()

      • Method getDeclaredMethod(方法名,参数列表)

    • 4、获取类名

      • String getName() 获取类名

  • 对成员变量Field的操作：

    • 设置值 void set(对象, 值)
    • 获取值 Object get(对象)

  • 对构造方法Constructor的操作：

    • 创建对象： constructor.newInstance(参数列表)
      • 如果使用空参数构造方法创建对象，操作可以简化：Class对象.newInstance()

  • 对成员方法Method的操作：

    • 执行方法： method.invoke(对象，参数列表)
    • 获取方法名：method.getName()

注解

1、JDK 中预定义的一些注解

  • @Override：用来检测被该注解标注的方法是否是继承自父类的

  • @Deprecated：表示该注解标注的内容，已过时

  • @SuppressWarnings：压制警告. 一般传递参数 all

2、自定义注解

  • 格式：

    • 元注解
    • public @interface 注解名称 {}

  • 本质：注解本质上就是一个接口，该接口继承java.lang.annotation.Annotation

  • 属性：接口中的抽象方法

    • 要求：

      • 1、属性的返回值类型有下列5种

        • 1、基本数据类型
        • 2、String
        • 3、枚举
        • 4、注解
        • 5、以上类型组成的数组

      • 2、定义了属性，在使用时需要给属性赋值

        • 1、如果定义属性时，使用default关键字给属性默认初始化值，，则使用注解时，可以不给这个属性赋值

        • 2、如果只有一个属性需要赋值，并且属性的名称是value，则value可以省略

        • 3、数组赋值时，值使用大括号 {} 包裹。如果数组中只有一个值，则 {} 可以省略

  • 元注解：用于描述注解的注解

    • @target：描述注解能够作用的位置

      • 类ElementType.TYPE、方法ElementType.METHOD、成员变量ElementType.FIELD、…

    • @Retention：描述注解被保留的阶段

      • RetentionPolicy.SOURCE 表示当前被描述的注解,不会保留到class字节码文件中
      • RetentionPolicy.CLASS 表示当前被描述的注解，会保留到class字节码文件中，不会被JVM读取到
      • RetentionPolicy.RUNTIME 表示当前被描述的注解，会保留到class字节码文件中，并被JVM读取到

    • @Documented：描述注解是否被抽取到API文档中

    • @Inherited：描述注解是否被子类继承

3、解析注解：获取注解中定义的属性值

  • 1、获取注解定义位置的字节码对象（Class，Method，Field）

  • 2、调用.getAnnotation(注解.class)获取注解对象

  • 3、通过注解对象.抽象方法名得到属性值

JavaWeb

1、Web相关概念

1.1、软件架构

  • C / S 客户端/服务器端
  • B / S 浏览器/服务器端

1.2、资源分类

  • 静态资源：所有用户访问后，得到的结果都是一样的，称为静态资源，静态资源可以直接被浏览器解析
  • 动态资源：每个用户访问相同资源后得到的结果可能不一样，称为动态资源，动态资源被访问后需要先转换为静态资源，再响应给浏览器
    • servlet、jsp、php…

1.3、网络通信三要素

  • IP：电子设备（计算机）在网络中的唯一标识
  • 端口：应用程序在计算机中的唯一标识 【0 ~ 65536】
  • 传输协议：规定了数据传输的规则
    • 基础协议：
      • TCP：可靠协议，三次握手，速度稍微慢
      • UDP：不可靠协议，速度快

1.4、web服务器软件

  • 服务器：安装了服务器软件的计算机
  • 服务器软件：接收用户的请求，处理请求，做出响应

在web服务器软件中，可以部署项目，让用户通过浏览器来访问这些项目

  • 常见的java相关的web服务器软件：
    • webLogic
    • webSphere
    • JBOSS
    • Tomcat

3、Servlet 【server applet】

3.1、Servlet 概念

定义：运行在服务器端的小程序

Servlet 就是一个接口，定义了Java 类被浏览器访问到（tomcat识别）的规则

将来自定义一个类，实现Servlet 接口，复写方法

3.2、Servlet 快速入门

在JavaEE 项目中，定义一个类，实现Servlet 接口，实现接口中的5个方法，

然后在web.xml 中配置Servlet ，配置url 访问路径

3.3、Servlet 执行原理

  • 当服务器接收到客户端浏览器的请求后，会解析请求URL 路径，获取访问的Servlet 的资源路径

  • 通过资源路径去web.xml 中找对应的 <url-pattern> 标签体内容 ，然后再找到<servlet-class> 全类名

  • tomcat 会将全类名对应的字节码文件加载进内存。 Class.forName(全类名);

  • 创建对象 cls.newInstance();

  • 调用service() 方法

3.4、Servlet 的生命周期

  • 1、被创建：执行init 方法，只执行一次

    • 默认情况下，第一次被访问时创建Servlet

    • 可以配置指定Servlet 的创建时机：在web.xml 中的<servlet> 标签中可以通过指定<load-on-startup> 的值

    • Servlet 的init 方法只执行一次，说明一个Servlet 在内存中只存在一个对象，Servlet 是单例的

      • 多个用户同时访问时，可能存在线程安全问题
        • 解决：尽量不要在Servlet 中定义成员变量；即使定义了成员变量，也不要对其修改值

  • 2、提供服务：执行service 方法，执行多次

    • 每次访问Servlet 时，Service 方法都会被调用一次

  • 3、被销毁：执行destroy 方法，只执行一次

    • Servlet 被销毁时执行。服务器正常关闭时，Servlet 被销毁。
    • 只有服务器正常关闭时，才会执行destroy 方法
    • destroy 方法在Servlet 被销毁之前执行，一般用于释放资源

步骤：

  • 2、在类上使用@WebServlet 注解配置

1、IDEA 会为每一个Tomcat 部署的项目单独建立一份配置文件

Servlet 接口 —–> GenericServlet 抽象类 —-> HttpServlet 抽象类

  • HttpServlet 抽象类：对http 协议的一种封装，简化操作

4、HTTP

4.1、HTTP 概念

HTTP：超文本传输协议

  • 传输协议：定义了客户端和服务器端通信时，发送数据的格式
  • HTTP 协议基于TCP/IP 的高级协议
  • HTTP 的默认端口号是 80
  • HTTP 是基于请求/响应模型的
  • HTTP 协议是无状态的：每次请求之间相互独立，不能交互数据

  • Get 请求的url 长度有限制；Post请求的url 长度没有限制
4.2.2、常见的请求头
  • User-Agent：浏览器告诉服务器，当前请求访问你使用的浏览器版本信息
  • Referer：告诉服务器，当前请求从哪里来

5、Request

实现Servlet 接口，浏览器发送请求流程

  • 1、Tomcat 服务器会根据请求url 中的资源路径，创建对应的对象
  • 2、Tomcat 服务器会创建request 和response 对象，request 对象中封装请求消息数据。
  • 3、Tomcat 服务器将request 和response 对象传递给service 方法，并调用service 方法
  • 4、程序员可以通过request 对象获取请求消息数据，通过response 对象设置响应消息数据
  • 5、Tomcat 服务器在给浏览器做出响应之前，会从service 方法中的response 对象中拿出设置的响应消息数据

request 对象和response 对象是由服务器创建的

5.1、request 功能

5.1.1、获取请求消息数据
  • 获取请求行数据
    • 获取请求方式：String getMethod();
    • 获取虚拟目录：String getContextPath();
    • 获取Servlet 路径：String getServletPath();
    • 获取get方式的请求参数：String getQueryString();
    • 获取请求的uri：String getRequestURI();
    • 获取请求的url：StringBuffer getRequestURL();
    • 获取协议及版本：String getProtocol();
    • 获取客户机的IP地址：String getRemoteAddr();
  • 获取请求头数据
    • 通过请求头的名称获取请求头的值：String getHeader(String 请求头名称);
    • 获取所有的请求头名称：Enumeration<String> getHeaderNames();
  • 获取请求体数据
    • 获取流对象，再从流对象中拿数据
    • 获取字符输入流【只能操作字符数据】：BufferedReader getReader();
    • 获取字节输入流【可以操作所有类型的数据】：ServletInputStream getInputStream();
5.1.2、其它功能

  • 1、获取请求参数：

    • 根据参数名称获取参数值：String getParameter(String 参数名);
    • 根据参数名称获取参数值的数组：String[] getParameterValues(String 参数名);
    • 获取所有请求参数的参数名称：Enumeration<String> getParameterNames();
    • 获取所有参数的Map集合：Map<String, String[]> getParameterMap();

    中文乱码问题：

    ​ get请求不会乱码

    ​ post请求乱码解决：在获取请求参数前，设置编码：httpServletRequest.setCharacterEncoding("utf-8");

  • 2、请求转发：【一种在服务器内部的资源跳转方式】

    • 1、通过request 对象获取请求转发器对象：RequestDispatcher getRequestDispatcher(String 路径)
    • 2、使用RequestDispatcher 对象进行转发：void forward(ServletRequest request, ServletResponse response)

    请求转发特点：

    ​ 1、浏览器地址栏路径不发生变化

    ​ 2、只能转发到当前服务器内部资源中

    ​ 3、转发只会发送一次请求

  • 3、共享数据：

    • 域对象：一个有作用范围的对象，可以在域范围内共享数据
      • request 域：代表一次请求，一般用于请求转发的多个资源中共享数据
    • 方法：
      • 设置属性：void setAttribute(String 属性名, Object 属性值);
      • 获取属性：Object getAttribute(String 属性名);
      • 删除属性：void removeAttribute(String 属性名);

  • 4、获取ServletContext 对象：ServletContext getServletContext()

8、会话技术

  • 会话：一次会话中包含多次请求和响应
    • 一次会话：浏览器第一次给服务器资源发送请求，会话建立，直到一方断开位置
    • 会话的功能：在一次会话的范围内的多次请求间，共享数据

8.1、Cookie 客户端会话技术

8.1.1、概念

客户端会话技术，将数据保存到客户端

8.1.2、使用步骤

  • 创建Cookie对象，绑定数据：new Cookie(String cookie名,String cookie值)

  • 发送Cookie对象：response.addCookie(Cookie cookie)

    服务器就会以：【 Set-Cookie：cookie名=cookie值 】 的形式在响应头中响应给浏览器

    设置完cookie 之后，在一次会话中浏览器再次发送请求时就会在请求头中带着cookie 发送给服务器【 cookie：cookie名=cookie值 】

  • 获取Cookie，拿到数据：request.getCookies()

8.1.3、Cookie 的细节

1、Cookie 在浏览器中的存活时间

  • 默认情况下，当浏览器关闭后，Cookie 数据被销毁
  • 可以设置Cookie 的生命周期：cookie.setMaxAge(int 秒数)
    • 正数：将Cookie 数据写到硬盘的文件中【也就是 持久化存储 】
    • 负数：默认值
    • 0：代表删除Cookie 信息

2、同一个服务器下，多个项目之间Cookie 的共享

  • 默认情况下，Cookie 不能共享
  • 设置Cookie 的获取范围：cookie.setPath(String 路径) 【默认情况下，cookie 的path 是 当前的虚拟目录 】
    • 如果要共享，可以将path 设置为 /
8.1.4、Cookie 的特点和作用
  • 特点：
    • Cookie 存储在客户端浏览器
    • 浏览器对于单个Cookie 的大小有限制，对同一个域名下的总Cookie 数量也有限制
  • 作用：
    • Cookie 一般用于存储少量的不太敏感的数据
    • 在不登录的情况下，完成服务器对浏览器的身份识别

8.2、Session：服务器端会话技术

8.2.1、概念

session：服务器端会话技术，在一次会话的多次请求间共享数据，将数据保存在服务器端的对象中。HttpSession

8.2.2、使用步骤

获取session 对象：request.getSession();

HttpSession 对象的方法：getAttribute(String name)、setAttribute(String name,String value)、removeAttribute(String name)

8.2.3、原理

当浏览器第一次向服务器端发送请求时，在服务器端内存中会生成一个session，服务器会在响应的时候把这个session 的id 通过Set-Cookie 传递给浏览器

浏览器就会把这个JSESSIONID 存储到cookie 中,当再次发送请求时，会在cookie 中携带JSESSIONID 发送给服务器，服务器就会根据Cookie 中的JSESSIONID 去内存中查找对应的Session 对象

session 的实现是依赖cookie 的

8.2.4、session 细节

  • 当客户端关闭后，服务器不关闭，两次获取的session 不是同一个

    • 默认情况下不是同一个，

    • 可以通过手动创建cookie 指定JSESSIONID 为session 的id，并设置cookie 的最大存活时间

  • 客户端不关闭，服务器关闭后，两个获取的session 不是同一个

    • session 不是同一个，但是要确保数据不丢失，就要用到session 的钝化
      • session 的钝化：在服务器正常关闭之前，将session 对象序列化到硬盘上
      • session 的活化：在服务器启动后，将硬盘上的session 文件转化成内存中的session 对象

  • session 的被销毁

    • 1、服务器关闭

    • 2、session 对象调用invalidate()

    • 3、session 的默认失效时间是30分钟

      • 可以在 /tomcat/conf/web.xml 中设置session 的失效时间

8.2.5、session 特点
  • session 用于存储一次会话的多次请求的数据，存在服务器端
  • session 可以存储任意类型，任意大小的数据
  • session 与cookie 的区别：
    • session 存储数据在服务器端，cookie 存储数据在客户端
    • session 没有数据大小限制，cookie 有
    • session 的数据安全，cookie 相对于不安全

9、JSP

9.1、概念

  • Java Server Page：Java服务器端页面
  • 可以理解为：一个特殊的页面，其中既可以定义html 标签，又可以定义java 代码
  • 用于简化书写

9.2、原理

  • JSP 本质上就是一个Servlet

9.3、JSP 的脚本

JSP 定义Java 代码的方式：

  • <% Java 代码 %>
    • 在service() 方法中定义的内容
  • <%! Java 代码 %>
    • 可以定义成员变量、成员方法
  • <%= Java 代码 %>
    • 在service() 方法中的输出语句，将Java 代码输出到页面上

9.4、JSP 内置对象

  • 概念：在jsp 页面中不需要获取和创建，可以直接使用的对象，就是内置对象
  • jsp 一共有9个内置对象：
    • 1、pageContext。【PageContext】 当前页面共享数据，还可以获取其它8个内置对象
    • 2、request。【HttpServletRequest】。一次请求访问的多个资源（请求转发）
    • 3、session。【HttpSession】。一次会话的多次请求间
    • 4、application。【ServletContext】。所有用户间共享数据
    • 5、response。【HttpServletResponse】。响应对象
    • 6、page。【Object】。当页面（Servlet）的对象this
    • 7、out：字符输出流对象，可以将数据输出到页面上。【JspWriter】 输出对象，将数据输出到页面
      • response.getWrite() 和out 的区别：
        • 在Tomcat 服务器给浏览器做出响应之前，会先找response 缓冲区数据，再找out 缓冲区数据
        • response.getWriter() 数据输出永远在out .write() 之前
    • 8、config。【ServletConfig】。Servlet 的配置对象
    • 9、exception。【Throwable】。异常对象

前4个为域对象

9.5、JSP 指令

  • 作用：用于配置JSP 页面，导入资源文件
  • 格式：<%@ 指令名称 属性名1=属性值1 属性名2=属性值2 %>
  • 分类：
    • page：用于配置JSP 页面
      • contentType：等同于resp.setContentType()
        • 可以设置响应体的mime 类型以及字符集
      • language：只有java
      • buffer：缓冲区
      • import：导包
      • errorPage：当前页面发生异常后，会自动跳转到指定的错误页面
      • isErrorPage：标识当前页面是否是错误页面
    • include：页面包含，导入页面的资源文件
    • taglib：导入资源

9.6、JSP注释

  • html 注释
    • <!-- --> 只能注释html
  • jsp 注释
    • <%-- --%> 注释所有

10、MVC 开发模式

  • M：Model，模型：处理具体的业务操作，如查询数据库、封装对象【JavaBean】
  • V：View，视图：展示数据 【Jsp】
  • C：Controller，控制器：获取请求参数；调用模型；将数据交给视图展示【Servlet】

浏览器发送请求到服务器，在服务器中控制器进行处理，然后调用模型进行业务处理，模型处理完成之后发送给控制器，控制器再将数据发送给视图进行渲染，视图再返回给浏览器

11、EL 表达式

  • EL：Expression Language 表达式语言

  • 作用：可以替换和简化JSP 页面中Java 代码的编写

  • 语法：${表达式}

  • 注意：Jsp 默认支持el 表达式，

    • 如果要忽略el 表达式，可以配置isELIgnored='true'，可以忽略当前页面中所有的el 表达式
    • 也可以在el 表达式之前加上 \ ，只忽略一个el 表达式

  • 作用：

    • 1、运算
      • 支持的运算符
        • 算术运算符：+、-、*、/【div】、%【mod】
        • 比较运算符：>、<、==、>=、<=
        • 逻辑运算符：&&【and】、||【or】、!【not】
        • 空运算符：empty ,用于判断字符串、集合、数组的对象是否为null，并且长度是否为0
          • ${empty list} 判断是否为空
          • ${not empty list\str} 判断是否不为空
    • 2、获取值
      • el 表达式只能从域对象中获取值
      • 语法：
        • ${域名称.键名}
          • pageScope —> pageContext
          • RequestScope —> request
          • sessionScope —> session
          • applicationScope —> application【ServletContext】
        • ${键名}：表示依次从最小的域中查找是否有对应的值，直到找到为止

  • 隐式对象：在el 表达式中可以直接使用的对象，就是隐式对象

    • el 表达式中共有11个隐式对象
    • pageContext:
      • 获取其它8个内置对象

12、JSTL 标签

  • 概念：JavaServer Pages Tag Library，Jsp 标准标签库

    • 是由Apache 组织提供的开源的免费的Jsp 标签

  • 作用：用于简化和替换Jsp 页面上的Java 代码

  • 使用：

    • 1、导入jstl 相关Jar 包
    • 2、引入标签库：<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>

  • 常用的JSTL 标签

    • if：类似于Java 代码中的if

      • <c:if test="${not empty requestScope.list}">
    <h3>list不为空</h3>
</c:if>

    • foreach：类似于Java 代码中的for 循环

  • 业务逻辑层：处理业务逻辑

2.3、Filter 过滤器的生命周期方法

1、init()方法： 在服务器启动后，会创建Filter对象，然后调用init方法。 一般用于加载资源， 只执行一次。

2、doFilter()方法：每一次请求被拦截时，都会执行。执行多次

3、destroy()方法：在服务器关闭后，Filter对象被销毁。如果服务器是正常关闭，则会执行destroy方法。用于释放资源， 只执行一次

2.4、Filter 过滤器配置详解

  • 拦截路径配置：

    • 1、具体的资源路径。比如 /index.jsp 只有访问index.jsp资源时，过滤器才会被执行
    • 2、具体的拦截目录。比如 /user/* 访问 /user 下的所有资源时，过滤器都会被执行
    • 3、后缀名。比如 *.jsp 访问所有的后缀名为jsp资源时，过滤器都会被执行
    • 4、拦截所有资源。 /* 访问所有资源时，过滤器都会被执行

  • 拦截方式配置:资源被访问的方式

    • 注解配置：

      • 设置dispatcherTypes属性：

        • 1、REQUEST：默认值。浏览器直接请求资源
        • 2、FORWARD：转发访问资源
        • 3、INCLUDE：包含访问资源
        • 4、ERROR：错误跳转资源
        • 5、ASYNC：异步访问资源

2.5、过滤器链（配置多个过滤器）
  • 执行顺序：如果有两个过滤器：过滤器1 和 过滤器2，顺序如下：
    • 请求 -> 过滤器1 -> 过滤器2 -> 资源执行 -> 过滤器2 -> 过滤器1 ->浏览器
  • 过滤器的先后顺序问题：
    • 1、注解配置：按照类名的字符串比较规则比较，值小的先执行
      • 如：AFilter 和 BFilter ，A比B小，所以AFilter先执行
    • 2、web.xml 配置：按照在web.xml 中定义的先后顺序，<filter-mapping>谁定义在上边，谁先执行
