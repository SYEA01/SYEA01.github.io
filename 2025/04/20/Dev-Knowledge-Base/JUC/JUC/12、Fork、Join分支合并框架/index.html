Fork/Join 框架 是 Java 7 引入的一种并行任务处理框架，专为 分治算法（Divide-and-Conquer） 设计，适用于能够递归拆分成子任务的大规模计算场景。其核心思想是 “任务拆分（Fork）” 和 “结果合并（Join）”，通过工作窃取（Work-Stealing）算法实现高效的并行计算。

1. 核心组件

(1) ForkJoinPool

  • 作用：管理线程和任务调度，是 ExecutorService 的实现类。
  • 特点：
    • 默认线程数 = CPU 核心数（可通过 -Djava.util.concurrent.ForkJoinPool.common.parallelism 调整）。
    • 使用 工作窃取算法：空闲线程主动从其他线程的任务队列尾部“窃取”任务执行，减少线程竞争。

(2) ForkJoinTask

  • 作用：表示可拆分的任务基类，具体实现包括：
    • **RecursiveTask<T>**：有返回值的任务（如计算总和）。
    • **RecursiveAction**：无返回值的任务（如数组排序）。

2. 工作流程

  1. 任务拆分（Fork）：将大任务递归拆分成小任务，直到达到阈值（如数据量足够小）。
  2. 并行执行：子任务由 ForkJoinPool 中的线程并行执行。
  3. 结果合并（Join）：将子任务的结果逐层合并，最终得到总结果。

3. 代码示例（计算数组元素和）

4. 关键机制

(1) 工作窃取（Work-Stealing）

  • 原理：每个线程维护一个双端队列（Deque），自己产生的任务从头部加入，其他线程从尾部窃取。
  • 优点：
    • 减少线程空闲：任务队列分布均匀。
    • 避免竞争：窃取操作发生在队列尾部，减少锁冲突。

(2) 任务拆分与合并

  • 阈值设置：任务拆分的最小粒度需平衡并行效率和任务管理开销。
  • 递归拆分：通过 fork() 异步提交子任务，join() 阻塞等待结果。

5. 适用场景

  • 分治算法：如归并排序、快速排序、矩阵乘法。
  • 大数据处理：统计、过滤、聚合操作。
  • 递归问题：树遍历、图搜索等。

6. 注意事项

(1) 避免过度拆分

  • 任务拆分过细会增加调度开销，降低性能。
  • 经验值：阈值应大于 1000 次简单操作。

(2) 避免阻塞操作

  • Fork/Join 框架假设任务是纯计算型，若任务包含 I/O 或锁等待，可能导致线程饥饿。

(3) 结果合并开销

  • 合并子任务结果时需确保逻辑高效，避免合并成为性能瓶颈。

7. 与普通线程池对比

特性 ForkJoinPool ThreadPoolExecutor
任务类型 可递归拆分的分治任务 独立任务
调度策略 工作窃取（任务队列双端操作） 固定队列（先进先出）
线程数优化 自动适配 CPU 核心数 需手动配置核心和最大线程数
适用场景 计算密集型分治任务 通用异步任务处理

8. Java 8+ 中的增强

  • 并行流（Parallel Stream）：底层使用 ForkJoinPool。
  • 公共池：通过 ForkJoinPool.commonPool() 获取全局共享池。

总结

Fork/Join 框架通过 递归任务拆分 和 工作窃取算法 显著提升了分治任务的并行效率，特别适合计算密集型的大规模数据处理。使用时需合理设置阈值，避免任务过细，并结合实际场景选择与普通线程池的搭配方案。
